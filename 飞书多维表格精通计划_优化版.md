# 飞书多维表格精通计划 — 优化版

## 设计原则

```
原版假设：你是一个没有编程基础、只是不熟多维表格的人
实际情况：你正在建立编程思维，同时从零学数据管理

原版逻辑：先学 → 再做 → 最后变现
优化逻辑：边拆（别人的）→ 边建（自己的）→ 能力溢出时自然变现

核心改动：
1. 时间按你的实际起点重估（不是理想状态）
2. 数据从你已有的真实资产入手（不编造）
3. 删掉变现时间线（换成能力信号）
4. 验收从"产出量"改为"限时挑战"
5. 第1周起加入"拆表"训练（不只是建）
```

## 每日节奏

| 时段 | 干什么 | 时长 | 关键区别 |
|------|--------|------|---------|
| 前20分钟 | **拆** — 打开一个飞书模板/别人的表，看结构、猜设计意图 | 20min | 原版没有这个环节 |
| 中间60分钟 | **建** — 在自己的真实项目中操作 | 60min | 不变 |
| 后20分钟 | **写** — 今天学到什么、卡在哪、明天干什么 | 20min | 压缩了，笔记不需要30分钟 |

**总计：每天约1.5小时，可执行性远高于每天2小时。**

**节奏弹性规则：**
- 状态好 → 中间环节可以拉到90分钟
- 状态差 → 只做"拆"的20分钟也算完成当天任务（保住连续性）
- 周末 → 可以做一次2-3小时的深度项目推进

## 阶段1：地基 — 数据思维建立（第1-4周）

### 目标

不是"学会多维表格的操作"，而是**建立"看到任何信息都能想到表结构"的思维习惯**。

---

### 第1周：认知翻转 + 第一张真实表

**学什么：**
- 字段类型的本质（文本/数字/日期/单选/多选/链接 = 数据的"形状"）
- 主键思维：每一行代表什么？用什么唯一标识它？
- 视图 ≠ 表（同一份数据，不同的看法）

**做什么：**
- Day1-2：把你浏览器书签导出，扔进一张多维表格，字段随便设
- Day3-4：看着这张"烂表"，想：哪些信息应该拆成独立字段？重新设计字段
- Day5-7：给这张表建3种视图（表格视图/看板视图/画册视图），体会"同数据不同视角"

**拆表练习：**
- 去飞书模板中心，打开"项目管理"类的任意模板
- 不要用它，只看它：有几个字段？为什么这样设？哪个是主键？

**本周结束时你应该能：**
- 看着任何一个Excel表说出"这里应该用单选而不是文本"
- 理解"表"和"视图"的区别（一句话说清）

---

### 第2周：关系的力量 — 一对多与关联

**学什么：**
- 一张表不够用的时候 = 你需要第二张表
- 关联字段：两张表之间的"桥"
- 一对多关系：一个分类 → 多条记录

**做什么：**
- 在你的学习资源表基础上，新建一张"来源渠道表"（公众号/YouTube/书/课程...）
- 用关联字段把每条资源链接到它的来源渠道
- 体会：现在你可以"按渠道"看所有资源了 — 这就是关系型数据库的力量

**拆表练习：**
- 打开飞书模板中心的"CRM客户管理"模板
- 画出它的表关系图：哪张表关联了哪张表？为什么？

**本周结束时你应该能：**
- 判断"这两个信息应该放一张表还是拆两张表"
- 用关联字段让两张表对话

---

### 第3周：筛选、分组、排序 — 数据的"问答"

**学什么：**
- 筛选 = 问数据库一个问题（"给我看所有关于Python的资源"）
- 分组 = 按某个维度汇总（"每个渠道有多少条资源？"）
- 排序 = 定优先级（"最近添加的排前面"）
- 组合使用 = 复杂查询的基础

**做什么：**
- 你的学习资源表现在应该有50+条真实数据了
- 创建5个"保存的视图"，每个对应一个你真实会问的问题：
  - "我还没看的收藏"
  - "最近一周新加的"
  - "关于某个特定主题的所有内容"
  - "按来源分组统计"
  - "标记为高质量的内容"

**拆表练习：**
- 找一个"OKR管理"模板，看它怎么用筛选和分组实现"本季度目标完成率"的展示

**本周结束时你应该能：**
- 任何人跟你说一个需求，你能立刻翻译成"筛选条件+分组方式+排序规则"
- 理解"视图"就是"保存的查询"

---

### 第4周：重构 — 推翻自己的第一版

**学什么：**
- 为什么第一版一定是错的（你的认知在进化，表结构必须跟着进化）
- 重构三问：冗余在哪？缺失在哪？关系对不对？
- 数据迁移的基本思路（旧表 → 新表，数据不丢）

**做什么：**
- 回头看你第1周建的表，此时你应该能看出至少3个问题
- 重新设计表结构（在纸上画）
- 执行重构：建新表 → 迁移数据 → 验证 → 删旧表
- 写一份"重构记录"：改了什么？为什么改？效果是什么？

**拆表练习：**
- 找一个你觉得"设计得不好"的模板（肯定能找到）
- 写出它的3个问题 + 你的改法

**本周结束时你应该能：**
- 对着自己的表说出"哪里不对"并动手改
- 这是你目标中"大刀阔斧改革"能力的第一次实战

---

### 阶段1验收（限时挑战）

```
挑战题：
给你一份杂乱的Excel表（50行，8列，有重复、有空值、有格式不统一）
要求：45分钟内
1. 判断应该拆成几张表
2. 在多维表格中建好结构
3. 导入数据并清理
4. 建出3个有意义的视图
5. 说出"如果数据量到5000行，当前结构撑不撑得住？为什么？"
```

## 阶段2：操控 — 公式与跨表联动（第5-8周）

### 目标

掌握"让数据自动计算、自动关联"的能力。不背公式，而是**建立"这个需求用什么公式"的直觉**。

---

### 第5周：基础公式 — 让字段自己算

**学什么：**
- IF / AND / OR — 条件判断（"如果状态是X，显示Y"）
- 数学类：SUM / AVERAGE / COUNT
- 文本类：CONCATENATE / LEFT / RIGHT / LEN
- 日期类：TODAY / DATEDIFF / DATEADD

**做什么：**
- 在你的学习资源表中加入公式字段：
  - "收藏天数"（TODAY() - 添加日期）
  - "是否过期未看"（IF 收藏超过30天且状态=未看，标红）
  - "资源标题+来源"自动拼接

**练习方法：**
- 不要背公式。遇到需求 → 先想"我要什么结果" → 再查"哪个公式能给我这个结果"
- 用AI辅助写公式，但你必须能**看懂**每一个参数的含义

**本周结束时你应该能：**
- 看到一个需求，3秒内判断"这需要公式"还是"这需要筛选"
- IF嵌套2-3层不慌

---

### 第6周：跨表公式 — 数据穿墙

**学什么：**
- LOOKUP / VLOOKUP — 从另一张表"拿"数据过来
- ROLLUP — 在关联字段上做聚合（求和、计数、平均）
- 关联字段 + ROLLUP = 跨表统计

**做什么：**
- 启动项目B：AI对话精华库
  - 表1：对话记录（日期、主题、关键内容、质量评分）
  - 表2：主题分类（主题名、描述、记录数[ROLLUP]、平均质量[ROLLUP]）
  - 表3：技能标签（标签名、关联记录数、最近更新时间）
- 实现："点开一个主题，自动显示它下面有多少条记录、平均质量多少"

**本周结束时你应该能：**
- 3张表互相关联、数据自动聚合
- 理解LOOKUP和关联字段的区别（什么时候用哪个）

---

### 第7周：复杂场景 — 嵌套与组合

**学什么：**
- 嵌套IF的实战场景（多条件优先级判断）
- SWITCH — 比嵌套IF更优雅的多分支
- 公式组合：IF + LOOKUP + 日期计算
- 错误处理：IFERROR（数据不完整时不要报错）

**做什么：**
- 在AI对话精华库中实现：
  - 自动打标签：根据内容关键词自动分类（用IF嵌套或SWITCH）
  - 优先级评分：根据"质量+时效+使用频率"三个维度算综合分
  - 过期提醒：超过N天没复习的高分内容标记为"该复习了"
- 数据量目标：表中应该有200+条真实数据

**本周结束时你应该能：**
- 写一个5层以内的复杂公式不查文档
- 看到别人的公式能快速读懂逻辑

---

### 第8周：大体量数据管理 + 性能思维

**学什么：**
- 数据量大了之后的问题：加载慢？查询卡？
- 字段设计对性能的影响（公式字段越多越慢）
- 归档策略：什么数据放"活跃表"、什么移到"归档表"
- 批量操作思维：不要一条条改，要想"怎么批量处理"

**做什么：**
- 把你两个项目的数据量推到500+
- 试一下：500条数据+10个公式字段，体感如何？
- 设计一套归档规则并执行
- 做一次"数据健康检查"：空值率、重复率、格式统一度

**本周结束时你应该能：**
- 对"大数据量下的表结构"有体感
- 知道什么时候该拆表、什么时候该归档

---

### 阶段2验收（限时挑战）

```
挑战题：
给你一个3张表的多维表格（有200+条数据），表之间已经有关联。
要求：40分钟内
1. 看懂现有结构和关联关系
2. 根据新需求（如"统计每个分类的平均完成时间"）写出公式
3. 发现结构中的一个设计问题并给出改进方案
4. 新建一个视图，用筛选+分组+公式展示一个"管理仪表盘"效果
```

## 阶段3：流动 — 自动化与脚本理解（第9-11周）

### 目标

让数据"自己动起来"。你的定位是**能看懂脚本、能指挥AI写脚本、能调试脚本**，不是从零手写。

---

### 第9周：飞书自动化（无代码）

**学什么：**
- 触发条件：什么事件发生时 → 执行什么动作
- 常见自动化模式：
  - 新记录创建 → 自动填充默认值
  - 字段变更 → 通知相关人
  - 定时触发 → 每天/每周自动执行某操作
  - 条件触发 → 当满足某条件时自动修改状态

**做什么：**
- 在你的学习资源表上配5条自动化：
  - 新增记录时自动填入当天日期
  - 标记为"重要"时自动发通知给自己
  - 每周一自动生成"本周待学习"视图的汇总
  - 状态改为"已完成"时自动计算耗时
  - 超过30天未处理的自动标记为"需清理"

**本周结束时你应该能：**
- 看到一个重复性操作就想"这能不能自动化"
- 配置任意触发条件+动作的组合

---

### 第10周：脚本认知 — 读懂 + 指挥AI写

**学什么：**
- 多维表格脚本能做什么（批量修改、数据清洗、外部数据导入）
- 脚本的基本结构（不需要你写，但需要你读懂）
- 怎么跟AI描述你的需求让它写出能用的脚本
- 怎么调试：报错了看什么、改什么

**做什么：**
- 用AI生成3个脚本并成功运行：
  - 批量清洗：把所有"文本格式的日期"转成"日期格式"
  - 批量关联：根据关键词自动匹配并建立关联
  - 数据导入：从一个CSV/JSON文件批量导入数据
- 每个脚本你要能：
  - 看着代码说出"第X行在做什么"
  - 改一个参数（比如筛选条件）并成功运行
  - 报错了能看懂错误信息并告诉AI怎么改

**本周结束时你应该能：**
- 区分"这个需求用自动化就够"和"这个必须用脚本"
- 给AI一个清晰的脚本需求描述，一次生成可用代码的成功率>70%

---

### 第11周：数据流设计 — 端到端

**学什么：**
- 数据从哪里来 → 怎么进入表 → 怎么流转 → 最终输出什么
- 命名规范：表名、字段名、视图名的统一规则（大体量必须有）
- 文档化：让别人（或未来的你）能看懂你的表设计
- 数据备份与版本概念

**做什么：**
- 为你的两个项目画"数据流图"：
  - 输入端：数据从哪来？手动录入/自动抓取/导入？
  - 处理端：进来后经过哪些自动化/公式处理？
  - 输出端：最终以什么形式被使用？视图/导出/API？
- 建立命名规范并执行（回去把所有不规范的名字改掉）
- 写一份"系统说明文档"：如果你把这个表交给另一个人，他能看懂吗？

**本周结束时你应该能：**
- 设计一个"不需要你在场别人也能用"的多维表格系统
- 这是从"个人工具"到"可交付产品"的关键跳跃

## 阶段4：实战 — 改革能力（第12-14周）

### 目标

你的终极目标——"拿到任何表都能大刀阔斧改革"。这个阶段全是实战。

---

### 第12周：诊断能力 — 快速看出问题

**做什么：**
- 从飞书模板中心下载5个不同领域的模板（HR/财务/项目管理/内容运营/客户管理）
- 每个模板用30分钟完成"诊断报告"：
  - 结构问题：冗余字段？缺失字段？关联设计不合理？
  - 性能问题：大数据量下会卡吗？公式有没有循环依赖？
  - 可扩展性：需求变了这个结构改得动吗？
  - 用户体验：视图设计合理吗？操作方便吗？

**本周结束时你应该能：**
- 打开任何一个多维表格，10分钟内说出至少3个结构性问题

---

### 第13周：重构执行 — 从诊断到落地

**做什么：**
- 从上周诊断的5个模板中选2个最烂的
- 完整重构：
  - 写重构方案（改什么、为什么改、预期效果）
  - 执行重构（保留原数据、建新结构、迁移、验证）
  - 写重构对比文档（Before vs After，用截图或描述）
- 另外找一个"真实的烂表"（可以是你自己之前做的、朋友的、网上的案例）做一次完整重构

**本周结束时你应该能：**
- 从"我看出问题了"到"我改好了"的完整闭环
- 重构过程中数据零丢失

---

### 第14周：综合实战 — 从零到交付

**做什么：**
- 模拟一个完整的"客户需求"：
  - 需求：一个小团队需要管理他们的内容创作流程（选题→写作→审核→发布→数据追踪）
  - 你要从需求分析 → 结构设计 → 建表 → 自动化 → 文档 → "交付"
- 限时：4小时内完成整个系统
- 交付物包括：
  - 多维表格本体
  - 使用说明文档
  - 数据流图
  - "未来可能的扩展方向"说明

**本周结束时你应该能：**
- 从一个模糊需求到一个可交付的完整系统
- 这就是你说的"精通"

---

### 阶段4验收（终极挑战）

```
终极挑战：
```

## 贯穿全程的"拆表"训练

| 周次 | 拆什么 | 关注什么 |
|------|--------|---------|
| 1-2 | 飞书模板中心的简单模板 | 字段设计、主键选择 |
| 3-4 | 中等复杂度模板（3张表以内） | 关联关系、视图设计 |
| 5-6 | 复杂模板（5张表+公式） | 公式逻辑、跨表数据流 |
| 7-8 | 找"设计得不好"的模板 | 问题诊断能力 |
| 9-11 | 带自动化的模板 | 自动化逻辑、触发条件 |
| 12-14 | 任何陌生表 | 全方位诊断+重构 |

每周至少拆1个，花20分钟。14周 = 至少拆14个不同的表。

## 与AI协作的正确姿势（贯穿全程）

```
阶段1-2（你在学基础时）：
  你问AI："这个字段应该用什么类型？"
  AI的角色：教练

阶段3-4（你有基础后）：
  你告诉AI："帮我写一个脚本，功能是XXX，输入是YYY，输出是ZZZ"
  AI的角色：执行者

最终状态：
  你做架构决策 → AI做执行 → 你验收
  你是导演，AI是演员
  你知道"要什么"和"对不对"，AI负责"怎么做"
```

## 与原版的核心区别总结

| 维度 | 原版 | 优化版 | 为什么改 |
|------|------|--------|---------|
| 总时长 | 12周 | 14周 | 给阶段1更多时间打地基 |
| 阶段1时长 | 2周 | 4周 | 数据思维需要沉淀，不能赶 |
| 数据来源 | 未明确 | 明确用已有真实数据 | 练思维而不是练操作 |
| 变现 | 写进时间线 | 改为能力信号 | 避免焦虑，能力到了自然发生 |
| 拆表训练 | 第10周才有 | 第1周就开始 | 不只建，还要能读和改 |
| 验收标准 | 产出计数 | 限时挑战 | 证明"会做"而不是"做过" |
| 脚本要求 | 你自己写 | 你能读+能指挥AI写 | 符合你实际起点 |
| 每日时长 | 2小时 | 1.5小时（弹性） | 可执行性优先 |
| 抗脆弱 | 无 | 状态差只做20分钟也OK | 保连续性比保强度重要 |

## 最后的话

这个计划的核心不是"14周后你变成专家"。

核心是：14周后，你拥有了一个**思维操作系统** ——

> 看到任何数据，你会自动想：该怎么结构化？
> 看到任何表，你会自动想：哪里可以更好？
> 遇到任何需求，你会自动想：该建几张表？怎么关联？怎么自动化？

这个思维操作系统一旦建立，无论后面你用飞书、用Notion、用Airtable、用SQL、甚至自己建数据库——底层都是一回事。

**你学的不是一个工具，你学的是一种思维方式。工具会过时，思维不会。**
