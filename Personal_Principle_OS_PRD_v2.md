## 一、系统定义

Personal Principle OS 是一个**个人决策辅助系统**：从真实决策行为中涌现底层原则，从外部高质量信息源中提取待验证假说，通过持续对比"你实际怎么做"和"你希望怎么做"之间的张力来驱动个人成长，并在每一个需要做判断的时刻提供基于你自己原则体系的双面论证辅助——而非替你做决策。

**它不是：** 知识库、笔记工具、格言收集册、检索式查阅手册、需要你打理的公司。
**它是：** 决策日志 + 假说验证引擎 + 原则涌现机制 + 对手盘决策辅助 + 认知演化追踪器。
**它像：** 定投纳斯达克——设定好规则后自动运行，你只在极少数结构性时刻介入。

### 系统核心价值锚定（v2新增）

系统存在的唯一理由：**帮你做出更好的决策。**

系统的不可替代价值集中在四件你用其他方式做不到的事：
1. **跨时间关联**——把你人脑无法同时hold住的、跨月跨年的决策数据关联起来
2. **强制对手盘**——在你所有朋友都可能顺着你说的情况下，结构性地给你反面论证
3. **无意识模式发现**——发现你完全没感觉的行为微趋势
4. **长期认知轨迹**——3年前怎么想、现在怎么想、变化的脉络

其他功能（记录、模式呈现、自我理解）系统都在做，但不是系统存在的理由——你用日记和自然反思也能部分做到。

### 系统终极目标（v2新增）

**系统的成功不是"被你永远使用"——而是"有一天你内化了足够的决策智慧，不再需要系统辅助也能高质量决策"。**

## 二、设计哲学（7条核心设计原则）

### 2.1 原则从行为中涌现，而非从外部灌入

系统不预设任何原则。第一阶段的核心任务是"无判断地记录真实决策"。原则是从你的决策模式中被系统识别出来的——它们是你"实际在遵循的东西"，可能跟你"想成为的人"完全不同。这个差距才是系统真正工作的地方。

外部信息源的内容不是原则，是"待验证假说"——只有经过你真实决策的检验才能升级为你的原则。

### 2.2 张力是核心价值，而非一致性

系统的核心输出不是"帮你做出正确的决策"，而是**持续暴露你的描述性原则（你实际怎么做）和规范性原则（你希望怎么做）之间的张力**。每一个张力都是一个成长点。系统帮你看见它、理解它、决定是接受它还是缩小它。

### 2.3 对手盘思维，而非确认偏误

系统在每一个重要决策中都主动扮演对手盘。它不告诉你"你应该选A"，而是呈现"基于你的原则体系，正面论证指向A，但反面论证（同样来自你的原则体系）指向B"。最终裁决永远是你做。

### 2.4 零摩擦输入，系统自治运行（v2修订）

你负责"说"（语音/文字，无格式要求），系统负责"想"（分类、结构化、关联、反馈）。

**v2关键变化：** 系统输出**默认生效**，你只在不同意时介入。不是"确认才生效"，是"生效了你要改随时改"。系统从"需要你管理的公司"变为"自己运行的指数基金"。

### 2.5 时间连续性：一切记录都有前因后果

没有任何一条记录是孤立的。每条决策都有回填追踪、每条原则都有生命周期、每条假说都有验证历史。

### 2.6 遗忘与增长同等重要（v2新增）

系统不是只有吸气没有呼气的肺。增长机制（记录、入库、生成）必须有同等强度的压缩/遗忘机制与之配对。复杂度有硬上限，周期性压缩是结构性的、强制性的、自动执行的。

### 2.7 系统对自身价值诚实（v2新增）

系统必须能被证明没用。如果运行数据表明系统没有改善决策质量，系统有义务告诉你。可证伪性不是弱点，是对你诚实的证明。

## 三、系统运行模型（v2新增）

### 3.1 系统成熟度曲线

```
用户认知投入
  ↑
  │  ████████████████████
  │  ██████████████████████████
  │  ████████████████████████████████████████████ ← 校准期结束
  │                                         ░░░░░░░░░░░░░░░░░░ ← 巡航态
  └──────────────────────────────────────────────────────────→ 时间
     校准期(30-60天)                          巡航期(永续)
```

### 3.2 三态运行模式

```yaml
system_states:
  calibration_mode:
    description: "用户教系统阶段"
    duration: "30-60天（直到所有确认行为毕业）"
    user_effort: "每天5-10分钟"
    behavior: "系统输出需确认，但每类确认有独立毕业条件"
    exit_condition: "所有确认行为的纠正率降到阈值以下"
    
  cruise_mode:
    description: "校准完成后的常态"
    user_daily:
      - "随时输入（想到就说，0摩擦，不需要确认）"
      - "需要时提问（决策咨询，主动发起）"
      - "固定时间看Briefing（<2分钟，≤5条）"
    user_monthly: "Sync Ritual（15分钟）"
    system_auto: "解析/分类/关联/检测张力/更新模式/压缩/遗忘"
    
  survival_mode:
    description: "用户状态差或高压期"
    trigger: "7天无输入 或 用户手动设置"
    behavior: 
      - "只存储输入，不做任何处理"
      - "不推送任何东西"
      - "不请求任何确认"
    recovery: "恢复输入后自动回升到cruise_mode，积压批量处理"
```

### 3.3 脉冲式运行节奏（v2新增）

系统活跃度随你的决策密度自动脉动，不是匀速运行。

```yaml
rhythm_detection:
  low_power_trigger: "连续14天输入<2条"
  full_power_trigger: "7天内输入≥5条 或 发起L5咨询 或 stakes=high"
  
  low_power_mode:
    - "不推送Briefing（除非你主动打开）"
    - "模式引擎暂停"
    - "只做：存储 + 长周期回填提醒"
    - "品质：完全安静，你感觉不到系统存在"
    
  full_power_mode:
    - "全层激活"
    - "Briefing恢复"
    - "主动关联推送"
    - "L5决策辅助全开"
    
  transition: "自动，不需要用户手动切换"
```

### 3.4 重激活校验协议（v2新增）

系统从低功耗切换到全功率时的第一个动作——校验而非直接调用旧数据。

```yaml
reactivation_protocol:
  trigger: "低功耗→全功率切换"
  first_action:
    - "取top 5最可能被调用的原则"
    - "检查每条原则的key_assumptions"
    - "对用户提一个问题"
  question: |
    距离上次活跃已经过了X个月。
    这些是你当时最核心的3条原则：
    ① [原则1] —— 前提是[情境A]
    ② [原则2] —— 前提是[情境B]
    ③ [原则3] —— 前提是[情境C]
    这些前提现在还成立吗？
  response_options:
    "是": "正常激活，全部原则可用"
    "部分变了": "标记变化原则降权使用，密集期后重新审视"
    "完全不同了": "所有原则标记待重新验证，进入重校准模式"
```

### 3.5 确认机制毕业条件（v2新增）

校准期内，每类确认行为有独立的退出跑道：

| 确认行为 | 毕业条件 | 毕业后行为 |
|---------|----------|-----------|
| 决策记录结构化 | 连续20条未修正 | 默认生效，不再呈现确认 |
| 维度分类 | 某Domain内连续10条未改 | 该Domain自动分类 |
| 试金石格式化 | 连续15条未调格式 | 默认入库 |
| 模式识别 | 前3个confirmed模式均认可 | 后续emerging直接标记 |
| TFRF评分 | 手动调分<2次/月 | 系统评分即终稿 |

**毕业不需要你手动操作——系统根据纠正率自动判断"我学够了"。**

| L4 → L5 | active张力列表（决策辅助盲区提醒） |

| L4 → 交互层 | 张力报告（汇入Briefing） |
| 全层 → L6 | 所有变化事件，供演化追踪 |

## 五、各层详细设计

### Layer 0: 决策日志（Decision Log）

**定位：** 整个系统的数据地基。你负责输入最少信息，系统负责补全和推断剩余字段。**默认生效，不需要逐条确认。**

#### 数据结构

```yaml
decision_log_entry:
  id: "D-037"
  created_at: "ISO时间戳"
  raw_input: "原始语音转文字/直接文字"

  # 系统解析（默认生效，confidence<0.5时标记待确认）
  scenario: "场景一句话描述"
  options_identified:
    - id: "opt_1"
      description: "选项描述"
  chosen_option: "opt_X"
  stated_reason: "你说出的理由"
  parsing_confidence: 0.85  # v2新增：解析置信度

  # 系统推断
  decision_dimensions: ["AX-001", "AX-005"]
  emotional_signals:
    - type: "conflict|anxiety|excitement|calm|frustration"
      indicator: "触发判断的原文片段"
  confidence_level: "high|medium|low|conflicted"
  reversibility: "high|medium|low"
  time_pressure: "high|medium|low"
  stakes_level: "trivial|low|medium|high|critical"
  decision_speed: "immediate|same_day|days|weeks|months"

  # 自动关联
  related_decisions: ["D-012", "D-028"]
  related_hypotheses: ["T-023"]
  related_principles: ["PP-003"]

  # 行为结构特征（v2新增：用于双通道模式检测）
  behavior_structure:
    effort_level: "high|medium|low"
    novelty: "routine|familiar|novel"
    choice_pattern: "status_quo|change|escalation|de-escalation"

  # 信任圈外部视角
  external_perspectives:
    - source: "人名"
      type: "casual_conversation|lifeline"
      content: "观点内容"

  # 回填追踪（v2修订：按stakes分级）
  outcome:
    status: "pending|positive|negative|neutral|mixed"
    backfill_schedule:  # v2：按stakes自动设定
      # stakes_trivial: no_backfill
      # stakes_low: [7_days]
      # stakes_medium: [30_days]
      # stakes_high: [30_days_feeling, 90_days_assessment]
      # stakes_critical: [30_days_feeling, 90_days, 180_days, 365_days]
    evaluations:
      - recorded_at: "日期"
        type: "feeling_only|preliminary|assessment|final"  # v2新增
        assessment: "positive|negative|neutral"
        negative_nuance: ""  # v2新增：即使正向，有没有小的负面后果
        notes: "描述"
    validates_hypotheses: []
    challenges_hypotheses: []

  # 元数据
  input_method: "voice|text"
  completeness: "full|partial"
```

#### 前置质量门控（v2新增）

系统不是"做了你再检查"，而是"确定度不够就老实说不确定"：

```yaml
quality_gates:
  decision_record:
    minimum_fields: [scenario, choice, reason]  # 少于3项→系统追问而非猜
    ambiguity_threshold: 0.5  # 解析confidence<0.5→标记"待确认"
    emotion_inference: "only_when_obvious"  # 不确定标"未知"
```

#### 输入处理流水线

```
用户原始输入
  → Step 1: 分类（决策记录 / 信息源触发 / 闪念）
  → Step 2: 结构化提取（场景/选项/选择/理由）
  → Step 3: 维度推断 + 行为结构特征提取
  → Step 4: 历史关联
  → Step 5: 默认生效写入（confidence≥0.5时）
  → Step 6: confidence<0.5的字段标记待确认，汇入Briefing
```

#### 决策日志压缩周期（v2新增）

```yaml
log_compression:
  cycle: "每90天执行"
  process:
    1: "识别已被模式引擎消化且产出confirmed原则的决策"
    2: "这些决策→压缩为一行摘要（日期+选择+结果）"
    3: "原始详细记录→删除（不是归档，是删除）"
    4: "保留：未消化的 + 有回填标记的 + stakes≥high的"
  purpose: "防止只增不减的膨胀"
```

#### 回填时间分级设计（v2新增）

```yaml
backfill_timing:
  stakes_trivial: no_backfill
  stakes_low: [7_days]
  stakes_medium: [30_days]
  stakes_high: [30_days_feeling_only, 90_days_real_assessment]
  stakes_critical: [30_days_feeling, 90_days_preliminary, 180_days_assessment, 365_days_final]

backfill_rules:
  high_and_critical_first_backfill:
    allowed_responses: ["记录当前感受", "记录客观变化"]
    NOT_allowed: ["正向/负向判断"]  # 太早判断被情绪污染
  every_positive_backfill:
    forced_question: "这个决策有没有任何小的负面后果，即使总体正向？"
    purpose: "强制收集负向信号碎片，防止正向回填过多"
```

### Layer 1: 试金石库（Touchstone Library）

**定位：** 存放外部信息源的观点，格式化为"待验证假说"。不是笔记本——每条内容都有生命周期，最终要么升级为原则，要么被推翻退役。

#### 数据结构

（保留v1完整数据结构，以下为关键字段）

```yaml
touchstone_entry:
  id: "T-089"
  created_at: "ISO时间戳"
  raw_input: "用户原始语音/文字"
  original_quote: "信息源原始表述"
  source: {person, platform, specific_content, date_consumed}
  your_reaction: "你为什么觉得有道理"

  hypothesis:
    statement: "可操作的假说表述"
    actionable_implication: "如果成立，在决策中意味着什么"
    trigger_scenarios: []
    boundary_conditions: {known: [], uncertain: []}
    quality_score:
      trigger: 0-3
      falsifiability: 0-3
      resolution: 0-3
      fidelity: 0-3
      total: 0-12
      grade: "A|B|C|D"

  lifecycle:
    status: "active_hypothesis|under_testing|partially_validated|validated|challenged|refined|retired|promoted"
    validation_events: []
    validation_score: {positive_count, negative_count, total_opportunities, utilization_rate}
    promotion_criteria: {min_positive_validations: 3, max_negative_ratio: 0.25, min_time_span_days: 30}

  relationships: {supports: [], conflicts_with: [], refines: []}
  domains: []
  hypothesis_level: "framework|derived|instance"
```

#### TFRF四维评分框架

| 维度 | 检验问题 | 3分 | 0分 |
|------|---------|-----|-----|
| T-Trigger | 什么场景想起它？ | 有场景+进入条件+排除条件 | 无触发 |
| F-Falsifiability | 什么结果能证明它错？ | 有明确失败条件 | 不可证伪 |
| R-Resolution | 能在几次决策中验证？ | ≤5次, ≤90天 | 不可验证 |
| F-Fidelity | 格式化后忠实原始逻辑？ | 保留核心推导+关键条件 | 歪曲原意 |

#### 复杂度硬上限（v2新增）

```yaml
touchstone_budget:
  active_hypotheses_hard_cap: 100
  at_cap_behavior: "TFRF评分最低的10条自动降级为archived"
  monthly_scan: "D级建议处理 / C级>60天未精炼降级 / 来源平均分趋势"
```

### Layer 2: 模式识别引擎（Pattern Engine）

**定位：** 后台运行，从决策日志中发现你未意识到的行为规律。★核心价值层——提供你自然反思做不到的跨时间统计发现。

#### 双通道模式检测（v2新增）

```yaml
pattern_detection_dual_channel:
  channel_1_reason_based:
    source: "stated_reason字段"
    method: "语义聚类，找重复逻辑"
    risk: "可能被事后合理化污染"
    
  channel_2_behavior_based:
    source: "选择本身的结构特征（behavior_structure字段）"
    method: "不看理由，只看选项客观属性"
    features: [reversibility, stakes, time_horizon, effort_level, novelty, choice_pattern]
    
  cross_validation:
    both_channels_agree: "高置信度模式"
    only_channel_1: "⚠️ 可能是合理化产物，降权处理"
    only_channel_2: "🔥 用户无意识的真实倾向——最高价值发现"
```

#### 模式类型

| 类型 | 定义 | 示例 |
|------|------|------|
| 偏好模式 | 某类场景中稳定偏好某种选择 | "金钱vs时间权衡时，80%选时间" |
| 情绪-行为模式 | 特定情绪与特定决策行为的关联 | "焦虑时决策速度快3倍，后悔率高" |
| 反模式 | 声称做A但实际反复做B | "说重视深度工作但62%时段选了碎片化" |
| 隐性模式（v2新增） | 行为通道发现但理由通道没有的 | "你总说选长期，但行为结构显示你80%选低effort" |

#### 模式呈现方式（v2修订）

**v1：** 呈现模式→用户确认。
**v2：** 呈现完整分布（含反例）→用户自判断。

```
不再说："发现模式：你倾向选长期价值。确认吗？"

而是说："在'短期vs长期'维度上，你过去30天的15条决策分布：
        · 12次选了长期方向（场景特征：经济压力低、时间充裕）
        · 3次选了短期方向（场景特征：deadline紧、情绪焦虑）
        
        你怎么理解这个分布？
        [这就是我的模式] [比这复杂] [我没意识到那3次]"
```

**前者引导确认，后者引导思考。**

#### 运行时机

| 频率 | 类型 | 做什么 |
|------|------|--------|
| 每条新决策 | 增量扫描 | 为已知模式提供新证据？ |
| 每周一次 | 全量扫描 | 新模式涌现？强度/趋势变化？ |
| 每月一次 | 深度分析 | 跨模式关联 + 双通道交叉验证 |

#### 边界条件

| 场景 | 处理 |
|------|------|
| 数据不够（<5条） | 不输出，宁缺毋滥 |
| 无稳定偏好（50/50） | 标记为"无模式" |
| 识别错误 | 用户否定。连续2次否定→停止追踪该候选 |
| 行为快速变化 | 检测"相变"→旧模式存档，重新积累 |

### Layer 3: 原则层（Principle Layer）

**定位：** 系统核心资产库。有层级、有关系、有生命周期的活体结构。

#### 原则类型

| 类型 | ID前缀 | 来源 | 含义 |
|------|--------|------|------|
| 描述性(Descriptive) | DP- | Layer 2模式涌现 | "你实际在做什么" |
| 规范性(Prescriptive) | PP- | Layer 1升级 + 主动声明 | "你希望怎么做" |
| 元原则(Meta) | MP- | 用户主动声明 | 最高层统摄，≤5条 |
| 已退役(Retired) | RP- | 曾是DP/PP，后被推翻 | "曾经信过但不再信" |

#### 数据结构

（保留v1完整数据结构，核心字段包括：statement, reasoning, operational_guide, boundaries, validation, hierarchy, relationships, weight, tension_sensitivity, context_validity, lifecycle）

#### 复杂度硬上限（v2新增）

```yaml
principle_budget:
  active_principles_hard_cap: 40
  at_cap_behavior: "新原则入库→系统必须同时提案一条旧原则退役/合并/降级"
  auto_merge_trigger:
    condition: "两条原则在同Domain + 验证历史70%重叠 + 核心逻辑可抽象涵盖"
    action: "系统提案合并为一条更高层原则"
    grace_period: "合并后6个月运行正常→旧两条真正删除"
```

#### 原则形成的4条通道

| 通道 | 路径 | 特征 |
|------|------|------|
| 1. 试金石升级 | 假说→多次验证→升级 | 有外部来源，经实践检验 |
| 2. 模式涌现 | 决策积累→模式确认→你认可 | 来自行为数据 |
| 3. 决策教训 | 极好/极差结果→提炼经验 | 来自深刻经历 |
| 4. 主动声明 | 你想清楚了→录入 | 初始unvalidated |

#### 冲突裁决机制

优先级体系：Tier 1(生存安全) > Tier 2(核心方向) > Tier 3(日常标准) > Tier 4(偏好)

同级冲突流程：明确冲突点→检查适用边界→检查前提假设→未消解则呈现用户裁决→同类冲突反复出现建议增加仲裁原则。

#### 情境依赖与有效性状态模型

```
情境不变 → VALID(高置信)
→ 情境重大变化 → SUSPENDED(存疑)
→ 新情境验证仍成立 → RE-VALID(跨情境稳定)
→ 需要修订 → REVISED
→ 不再适用 → RETIRED
```

### Layer 4: 张力检测器（Tension Detector）

**定位：** ★核心价值层。持续对比描述性原则和规范性原则，检测"你说的"和"你做的"之间的差距。提供你自然反思无法感知的偏离率数据。

#### 张力的精确定义

```
张力 = 在时间窗口T内，你的实际决策模式（描述性）与你声称希望遵循的标准（规范性）
       之间的累积偏离，达到了你自己设定的敏感度阈值。
```

#### 张力的四种来源类型

| 类型 | 定义 | 示例 |
|------|------|------|
| 行为-意图张力 | DP与PP指向相反 | "焦虑时快速决策" vs "不在情绪波动下做重大决策" |
| 频率-声明张力 | PP声称高频但实际使用率低 | 被触发12次，遵循4次 |
| 结果-信念张力 | PP一直遵循但结果持续为负 | 一直选长期但连续方向错 |
| 演化-锚定张力 | 行为已相变但PP还锚定旧模式 | 最有价值也最难检测 |

#### 张力呈现框架轮换（v2新增）

**不再固定用"偏离率"框架——随机轮换三种解读：**

```yaml
tension_framing:
  rotate_randomly_each_time:
    frame_A: "PP-003偏离率40%。你的行为在偏离你的原则。"
    frame_B: "你的行为在过去6周呈现新模式。PP-003可能不再描述真实的你。"
    frame_C: "数据显示PP-003和近期行为不一致。可解读为'偏离'或'原则过时'。你怎么看？"
  purpose: "防止用户内化'偏离=问题'的固定心智模型"
```

#### 复杂度硬上限（v2新增）

```yaml
tension_budget:
  active_tensions_hard_cap: 10
  at_cap_behavior: "最旧的未解决张力自动降级为observation"
  healthy_range: "2-4条active"
```

#### 检测引擎运行逻辑

（保留v1逻辑：即时扫描+偏离累积+阈值触发）

#### 用户响应选项

| 选择 | 动作 |
|------|------|
| 想缩小差距 | 进入行动计划 |
| 原则该更新 | 触发修订流程 |
| 加边界条件 | 更新boundaries |
| 误检 | 调整检测逻辑 |
| 以后再说 | 保持active，设defer |

### Layer 5: 决策辅助层（Decision Aid）

**定位：** ★核心价值层。系统存在的最核心理由——在决策发生之前提供结构化对手盘论证。这是你用其他方式最难做到的事。

#### 决策前直觉捕获（v2新增）

```yaml
pre_analysis_capture:
  trigger: "用户发起L5决策咨询时，系统呈现分析之前"
  question: "在你看系统分析之前——你现在倾向哪个选项？信心多大(1-10)？"
  fields:
    pre_lean: "选项X"
    pre_confidence: 7
  purpose:
    - "建立反事实基准（没有系统你会怎么选）"
    - "捕获直觉vs理性的真实张力"
    - "为系统价值追踪提供数据"
```

#### 推理引擎四阶段架构

```
用户输入场景 → Phase 1:场景解构 → Phase 2:原则召回 → Phase 3:论证合成 → Phase 4:盲区扫描
```

**Phase 1: 场景解构**
- 识别选择结构（binary/multi_option/spectrum/meta_decision）
- 提取维度、约束、时间尺度
- 时间尺度升维机制（反复同类微决策→建议升级到策略级）

**Phase 2: 原则召回——三层漏斗**
```
粗筛：维度匹配+trigger语义匹配 → 候选集
排序：relevance_score = 维度×0.3 + trigger×0.3 + tier×0.2 + validation×0.1 + recency×0.1
对抗性筛选：全指同一方向→强制拉入反方→没有则从假说找→标记盲区
```

**Phase 3: 论证链合成**
- 把多条原则串成推理链，每步标注依赖+confidence
- 正面论证：结论+reasoning_chain+overall_confidence+weakest_link
- 反面论证：结论+reasoning_chain+strongest_point
- 合成算法：识别锚点→检查前提→满足=支撑/不满足=攻击→连接扩展→标注confidence

**Phase 4: 盲区扫描**
- 缺失维度 / 认知偏误警报 / 信息缺口 / 活跃张力提醒 / 退役原则幽灵 / 预设验尸

#### Quick_test设计标准

三条件：一个问题 + 答案binary + 调用情绪直觉而非理性分析。
本质：把复杂推理压缩成5秒内回答的情绪探针。

#### 决策后闭环（v2修订）

```
决定后 → 写入L0 → 标记：
  - 调用了哪些原则
  - 是否遵循正面论证
  - pre_lean vs final_choice是否一致（v2新增）
  - 如果改变了选择→记录change_reason（v2新增：反事实追踪）
→ 设置回填（按stakes分级）
→ 如果选了对手盘方向→标记against_primary_recommendation
```

#### L5架构预留升级（v2新增）

当前：单线程线性推理。
v2.1预留方向（待实际验证后启用）：并行推理+讨论综合架构。

```
未来升级方向（不在v2.0实施）：
  并行阶段：视角A(原则出发) / 视角B(历史类比) / 视角C(反方) / 视角D(假说)
  讨论阶段：综合ABCD → 找共识/分歧/盲区 → 合成最终论证
  启用条件：系统运行3个月后，如果L5论证质量不够好（尤其对手盘太弱）
```

### Layer 6: 演化追踪层（Evolution Tracker）

**定位：** ★核心价值层。记录原则生命周期、认知变化轨迹。你用其他方式完全做不到的长期认知轨迹追踪。

三大功能模块：
- **模块A 演化叙事引擎**："你变了什么"
- **模块B 趋势预测引擎**："你即将变什么"
- **模块C 身份内核检测器**："什么没变"

#### 模块A：演化叙事引擎

把事件串成"你曾经认为X→因为Y发生了→你修正为Z"的叙事。
意义标注类型：progressive_deepening / oscillation / paradigm_shift / gradual_erosion / sudden_crystallization

#### 模块B：趋势预测引擎

6种预测信号：偏离加速 / 反例聚集 / 使用衰减 / 替代行为涌现 / 假说反超 / 元原则松动

价值：把认知滞后从3-6个月压缩到2-4周。

#### 模块C：身份内核检测器

身份内核 = 从未修改核心逻辑的原则 + 从未偏离的规范性原则 + 数据证明确实没变的行为倾向。

**成长vs漂移区分：**

| 维度 | 成长特征 | 漂移特征 |
|------|---------|---------|
| 驱动力 | 有明确触发事件 | "不知道什么时候开始的" |
| 方向性 | 精确化或深化 | 模糊化或放弃 |
| 意识度 | 你知道自己变了 | 系统指出后才发现 |
| 一致性 | 与内核不冲突 | 与内核产生新张力 |

#### 系统季度自我暴错（v2新增）

```yaml
quarterly_self_challenge:

  presentation: |
    本季度你有X次违背了系统的主要论证方向，其中Y次结果更好。
    系统在以下场景下判断力不如你的直觉：[...]
    请记住：系统不比你聪明，它只是更有结构。
  purpose: "主动去权威化，防止用户过度依赖系统"
```

## 六、维度体系设计

**定位：** 维度是系统的"语义路由表"——决定哪些决策被认为是"同一类"。

### 三层结构

```
第一层：维度域（Domain）— 预设≤10个 — 最粗归类，几乎不变
  例：时间分配/金钱资源/关系社交/职业方向/学习成长/健康生存/创作表达/决策方法论

第二层：维度轴（Axis）— 半涌现 — Domain内的具体张力轴
  例：深度vs碎片 / 当前任务vs新机会 / 为自己vs为他人

第三层：情境标签（Context Tag）— 完全涌现 — 只用于检索不用于分析
```

**聚类和模式发现只用第一层和第二层。**

### 语义匹配机制

**不依赖关键词，依赖"选择结构"匹配。** 两条决策"同一维度"的判定标准：面对的权衡结构相似（[放弃什么类型价值, 获得什么类型价值]）。

### 复杂度硬上限（v2新增）

```yaml
axis_budget:
  active_axes_hard_cap: 25
  at_cap_behavior: "触发合并扫描"
  merge_condition: "决策集合重叠>70% + 方向高度一致"
```

### 冷启动

- 前10天：预设3-5个Domain + 每个1-2个种子Axis
- 第11-20天：Domain内5+条决策 → 尝试Axis匹配
- 第21-30天：第一批涌现Axis可能出现

## 七、交互设计

### 7.1 交互场景（v2精简）

| # | 场景 | 触发方 | 频率 |
|---|------|--------|------|
| 1 | 捕获(Capture) | 用户 | 随时（想到就说） |
| 2 | 咨询(Consult) | 用户 | 重大决策时 |
| 3 | 每日Briefing | 系统 | 固定时间，有内容才推 |
| 4 | 复盘回填(Review) | 系统提醒 | 按stakes分级的时间点 |
| 5 | 深度探索(Explore) | 用户 | 按需 |
| 6 | 信任圈·日常沉淀 | 用户事后 | 线下交流后 |
| 7 | 信任圈·Lifeline | 用户(极稀有) | 每人每年1-2次 |
| 8 | 月度Sync | 系统+用户 | 每月一次15分钟 |

### 7.2 捕获（Capture）

**触发：** 快捷键/全局热键，从触发到输入<2秒
**输入：** 语音或文字，无格式要求
**系统处理：** 分类→结构化→关联→**默认生效写入**
**v2变化：** 不再呈现确认界面（校准期毕业后）。低confidence的汇入Briefing。

### 7.3 咨询（Consult）

**触发：** 主动切换到咨询模式
**v2新增：** 分析呈现前先捕获直觉倾向
**输出结构：**

```
🎯 决策咨询 #C-XXX
📋 场景解析：[核心选择/维度/时间压力/可逆程度]
⚖️ 正面论证：[推理链+每步依据+confidence+weakest_link]
🥊 对手盘：[反面推理链+strongest_point]
🔲 盲区扫描：[缺失维度/认知偏误/信息缺口/活跃张力/退役原则幽灵/预设验尸]
❓ 收敛问题：[帮你思考的问题]
📎 历史类比：[过去类似情况]
⚡ Quick Test：[5秒情绪探针]
→ 你的决定是？
```

### 7.4 每日Briefing（v2重设计）

**核心原则：没有重要信息的日子=不推送=最好的日子。**

| 优先级 | 内容类型 | 出现条件 |
|--------|---------|---------|
| 1 | 系统自我修改通知 | 系统改了自己的规则时 |
| 2 | 需要你输入的卡点 | 解析confidence<0.5的记录 |
| 3 | 发现/趋势摘要 | 有新emerging模式或显著趋势 |
| 4 | 即将到期的回填 | 回填点前1天 |
| 5 | 系统健康异常 | 指标超阈值 |

**约束：** 总条数≤5 / 每条一句话+可展开 / 总阅读<2分钟

### 7.5 月度Sync Ritual（v2精简）

**15分钟，固定时间：**

1. **系统健康一览**（2min）：核心指标+系统本月自动做了什么
2. **3条随机抽检**（5min）：系统随机呈现3条它自动处理的记录，你10秒扫一眼
3. **本月重大变更汇总**（5min）：系统升级了什么/新模式/新张力
4. **内核确认**（3min）：不变的内核→你确认

### 7.6 交互设计底层哲学

| 原则 | 含义 |
|------|------|
| 零摩擦捕获 | 想到→记录，无思考负担 |
| 默认生效 | 系统输出不等确认，你改随时改 |
| 永远透明 | 系统每个判断都可追问 |
| 你做决定 | 系统只呈现正反+你裁决 |
| 安静优先 | 没重要事不推送，稳定期完全沉默 |
| 对手盘思维 | 重要决策必有反面论证 |
| 脉冲式 | 系统活跃度随你的决策密度同步 |

## 八、信任圈机制

### 8.1 核心设计原则

**对方不需要碰你的系统，不需要知道系统存在。**

### 8.2 两种机制

| 机制 | 触发 | 对方感受 | 频率 |
|------|------|---------|------|
| 日常沉淀 | 线下交流后你事后录入 | 无感知 | 随时 |
| Lifeline | 重大决策+系统门槛通过 | 受宠若惊 | 极稀有,≤1-2次/年/人 |

### 8.3 日常沉淀

聊完回家录入。系统：结构化→关联决策→标记来源。对方负担：零。

伴侣特殊：面对面讨论后录入，可标记"联合决策"。

### 8.4 Lifeline机制

**稀缺性保护门槛：** 影响≥1年 + 不可逆高 + 距上次久 + 已独立思考>7天。
**系统帮你：** 生成精炼提问结构，让对方5分钟理解。
**回复处理：** 录入→结构化→关联→记录价值评估。

## 十、系统自主学习机制

### 10.1 双管线模型

你看信息源提取"对我决策有什么用"（内容层面）→产出假说。
系统看同一信息源提取"能不能优化我的运行逻辑"（元层面）→产出系统更新提案。

### 10.2 系统关注的五类知识

| 类型 | 找什么 | 示例 |
|------|--------|------|
| 思维框架 | 改进模式识别/推理逻辑 | OODA循环、贝叶斯更新 |
| 决策方法论 | 升级L5推理引擎 | 预设验尸法、10/10/10法则 |
| 认知偏误研究 | 加入盲区扫描 | 新发现的偏误和去偏方法 |
| 系统设计模式 | 优化架构本身 | 反脆弱设计、渐进复杂度管理 |
| 前沿研究 | 决策科学新发现 | 特定条件下某策略更优的实证 |

### 10.3 审批机制（v2修订）

**v1：** 每条提案都需用户逐条审批。
**v2：** 按scope分级自动处理：

```yaml
approval_by_scope:
  minor: "默认试运行14天，月度Sync时汇报结果"
  moderate: "默认试运行14天，月度Sync时决定保留或回滚"
  major: "推送审批，用户明确批准才执行"
  structural: "推送审批 + 详细论证 + 必须批准"
```

### 10.4 边界控制

```yaml
auto_learning_constraints:
  max_proposals_per_week: 3
  max_pending_updates: 5
  max_monthly_approved: 4
  post_update_observation: 14天
  all_updates_reversible: true
```

### 10.5 必须告知用户的底线（v2新增）

**无论scope大小，系统修改了自己的规则后，必须在下次Briefing中告知用户：**
- 改了什么
- 为什么改

- 怎么回滚

## 十二、冷启动计划

### Phase 1: 纯记录期（第1-10天）——校准期开始

- 每天记录决策（大小不限，有就记没有不强求）
- 格式极简：场景/选项/选择/理由
- 试金石库开始运行
- 系统不输出分析
- 维度体系：预设3-5个Domain + 种子Axis
- **每条记录需要确认**（校准系统理解力）

### Phase 2: 模式识别期（第11-20天）

- 系统对前10天记录跑第一轮分析
- 检测：倾向性？情绪-决策关联？理由中的重复逻辑？
- 双通道交叉验证开始
- 可能产生第一个emerging模式
- **确认行为开始逐项毕业**

### Phase 3: 初步涌现期（第21-30天）

- 系统输出第一批描述性原则（3-5条）
- 呈现完整分布（含反例）让你判断
- 维度体系：第一批涌现Axis可能出现
- **大部分确认行为应已毕业→进入巡航态**

### Phase 4: 巡航态确认（第31-60天）

- 验证系统默认处理的质量
- 月度Sync第一次执行
- 过程指标建立基线
- **所有确认行为毕业→正式进入cruise_mode**

## 十三、设计决策记录（Design Decision Log）

| 决策点 | 最终选择 | 否决方案 | 原因 |
|--------|---------|---------|------|
| 冷启动策略 | 先记录30天再涌现原则 | 从信息源直接提炼 | 别人的原则未经验证不是你的 |
| 信息源定位 | 待验证假说 | 直接作为原则来源 | 防止确认偏误引擎 |
| 决策辅助输出 | 对手盘(正反两面) | 给出确定结论 | 确认偏误风险 |
| 信任圈交互 | 你事后录入 | 对方在系统中操作 | 对方无动机 |
| Lifeline设计 | 极稀有+门槛保护 | 随时可问 | 稀缺性=社交价值 |
| 原则类型 | 描述性+规范性+张力 | 单一类型 | 张力才是成长引擎 |
| 张力定义 | 行为累积偏离 | 原则文本对比 | 文本矛盾≠实际张力 |
| 论证合成 | 推理链合成 | 列举各原则 | 列举式还得自己整合 |

| 情境衰减 | 情境相变跳转 | 线性时间衰减 | 时间不影响有效性 |
| 系统学习 | 双管线(内容+元) | 单管线 | 系统本身也需演化 |
| **确认机制（v2）** | **默认生效+毕业条件** | **每条确认才生效** | **认知负荷诚实性** |
| **复杂度管理（v2）** | **硬上限+强制遗忘** | **软建议** | **防止自催化膨胀** |
| **运行节奏（v2）** | **脉冲式自动切换** | **固定频率** | **匹配人生决策密度** |
| **系统价值验证（v2）** | **Kill Switch+反事实追踪** | **无验证机制** | **可证伪性=诚实** |
| **权力关系（v2）** | **主动去权威化** | **被动Override** | **防止了解变束缚** |
| **模式检测（v2）** | **双通道（理由+行为结构）** | **单通道（仅理由）** | **防事后合理化污染** |
| **系统终极目标（v2）** | **让你不再需要系统** | **永续使用** | **工具不是信仰** |
