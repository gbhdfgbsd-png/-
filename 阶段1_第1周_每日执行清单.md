# 阶段1 · 第1周：认知翻转 + 第一张真实表 — 每日执行清单

> 本文档是[飞书多维表格精通计划_优化版.md](./飞书多维表格精通计划_优化版.md)的子文件
> 覆盖范围：阶段1 第1周（Day 1 ~ Day 7）
> 每日时间预算：约1.5小时（20分钟拆 + 60分钟建 + 20分钟写）

---

## 本周目标

- 建立"字段类型 = 数据形状"的认知
- 理解主键思维：每一行代表什么？用什么唯一标识它？
- 体会"视图 ≠ 表"：同一份数据，不同的看法
- 完成第一张基于真实数据的多维表格

---

## 本周核心概念速查

| 概念 | 一句话解释 |
|------|-----------|
| 字段类型 | 文本/数字/日期/单选/多选/链接 = 数据的"形状"，决定了数据能怎么被查询和计算 |
| 主键 | 唯一标识一行记录的字段，回答"这一行到底是什么" |
| 视图 | 同一张表的不同"看法"，数据没变，展示方式变了 |
| 表格视图 | 最基础的行列展示，类似Excel |
| 看板视图 | 按某个单选/状态字段分列展示，适合流程管理 |
| 画册视图 | 卡片式展示，适合有封面图/摘要的内容浏览 |

## Day 1（周一）—— 原始数据导出 + 初识多维表格

### 拆（20分钟）

- [ ] 打开飞书多维表格模板中心
- [ ] 找到任意一个"项目管理"类模板并打开
- [ ] 回答以下问题（写在笔记中）：
  - 这个模板有几个字段？
  - 每个字段分别是什么类型？（文本？单选？日期？）
  - 哪个字段是"主键"（唯一标识每一行的）？
  - 为什么设计者选了这些字段？

### 建（60分钟）

- [ ] 导出你的浏览器书签（Chrome: 书签管理器 → 右上角 → 导出书签）
- [ ] 在飞书中创建一个新的多维表格，命名为"学习资源库 v1"
- [ ] 随意建几个字段（不用纠结对错），比如：
  - 标题（文本）
  - 链接（链接）
  - 备注（文本）
- [ ] 手动录入 **10条** 你最近收藏的学习资源（文章/视频/教程）
- [ ] 感受一下：哪些信息你想记录但现在的字段装不下？

### 写（20分钟）

- [ ] 记录今天的操作步骤和遇到的问题
- [ ] 写下你觉得"这个表还缺什么"的想法
- [ ] 明天的目标：继续导入数据，开始思考字段要不要调整

## Day 2（周二）—— 数据灌入 + 体感"字段不够用"

### 拆（20分钟）

- [ ] 再打开一个不同的模板（比如"读书笔记"或"内容管理"类）
- [ ] 对比昨天看的模板，回答：
  - 两个模板的字段设计有什么共同点？
  - 有什么明显不同？为什么不同？（业务不同 → 字段不同）
  - 哪个模板的字段类型用得更"讲究"？

### 建（60分钟）

- [ ] 继续往你的"学习资源库 v1"中录入数据
- [ ] 目标：今天结束时表中有 **25-30条** 记录
- [ ] 录入过程中注意观察：
  - 有没有字段你反复填一样的内容？（→ 可能应该是"单选"）
  - 有没有一个字段里你想填多个值？（→ 可能应该是"多选"）
  - 有没有信息你想记录但不知道放哪？（→ 可能缺字段）
- [ ] 把这些观察记下来，**先不改表结构**（忍住，Day3再改）

### 写（20分钟）

- [ ] 列出你发现的所有"不舒服的地方"（至少3个）
- [ ] 尝试回答：如果重来一次，你会怎么设计这些字段？
- [ ] 明天的目标：正式重新设计字段结构

## Day 3（周三）—— 字段重设计：从"随便填"到"结构化"

### 拆（20分钟）

- [ ] 打开你Day1看的项目管理模板
- [ ] 这次关注**字段类型的选择**：
  - 哪些字段用了"单选"而不是"文本"？为什么？
  - 有没有"日期"类型的字段？它解决了什么问题？
  - 有没有"人员"或"关联"类型的字段？
- [ ] 思考：如果这些字段全部换成"文本"，会出什么问题？

### 建（60分钟）

- [ ] 根据Day2的观察，**重新规划你的字段**，建议结构：

  | 字段名 | 类型 | 用途 |
  |--------|------|------|
  | 资源标题 | 文本 | 主键，唯一标识 |
  | 链接 | 链接 | 原始URL |
  | 资源类型 | 单选 | 文章/视频/教程/书/工具 |
  | 主题标签 | 多选 | Python/AI/设计/效率/... |
  | 来源 | 单选 | 公众号/YouTube/B站/Twitter/... |
  | 添加日期 | 日期 | 什么时候收藏的 |
  | 状态 | 单选 | 未看/在看/已看/已弃 |
  | 质量评分 | 数字(1-5) | 看完后打分 |
  | 笔记摘要 | 文本 | 一句话核心收获 |

- [ ] 在多维表格中修改/新增字段
- [ ] 把已有的30条数据按新字段补全信息
- [ ] 体会：结构化之后，数据是不是"清爽"多了？

### 写（20分钟）

- [ ] 对比 v1 结构和修改后的结构，写下"改了什么、为什么"
- [ ] 标记：哪些字段类型你之前不知道可以这样用？
- [ ] 明天的目标：继续填充数据 + 体验"有结构之后录入变快了"

## Day 4（周四）—— 数据填充 + 主键思维强化

### 拆（20分钟）

- [ ] 去模板中心找一个"CRM客户管理"或"人事管理"类模板
- [ ] 重点观察"主键"的设计：
  - 它用什么字段唯一标识每条记录？（客户名？编号？）
  - 如果两个客户同名怎么办？它怎么处理？
  - 这个模板里有没有"自动编号"字段？
- [ ] 想一想：你自己的表，"资源标题"够不够当主键？万一标题重复呢？

### 建（60分钟）

- [ ] 继续往表中添加数据，目标今天结束时有 **40-50条**
- [ ] 操作中注意：
  - 利用"单选"的下拉快速选择，体会比打字快多少
  - 利用"多选"给资源打多个标签
  - 尝试用"日期"字段的日期选择器
- [ ] 思考并决定：要不要加一个"自动编号"字段作为真正的唯一标识？
- [ ] （可选）如果你发现需要新的单选选项（比如新的"资源类型"），直接加

### 写（20分钟）

- [ ] 记录：结构化之后，录入速度是不是变快了？
- [ ] 回答："什么是好的主键？"（用自己的话，一句话）
- [ ] 明天的目标：学会视图，让同一份数据用不同方式展示

## Day 5（周五）—— 视图入门：表格视图 + 看板视图

### 拆（20分钟）

- [ ] 回到你之前看过的任意一个模板
- [ ] 这次关注**视图**：
  - 这个模板提供了几种视图？
  - 每种视图用了什么筛选/分组/排序？
  - 为什么同一个表需要多种视图？（不同角色？不同场景？）
- [ ] 关键认知：**视图不会改变数据本身，只改变"看数据的方式"**

### 建（60分钟）

- [ ] 在你的学习资源表中创建 **视图1：默认表格视图**
  - 按"添加日期"降序排列（最新的在最上面）
  - 这就是你的"时间线视图"

- [ ] 创建 **视图2：看板视图（按状态分列）**
  - 分组字段选"状态"
  - 你会看到4列：未看 / 在看 / 已看 / 已弃
  - 试着拖拽一条记录从"未看"到"在看" → 数据自动变了！
  - 体会：看板 = 可视化的工作流

- [ ] 操作确认：
  - 在看板视图里改了状态，切回表格视图看看 → 数据确实变了
  - 这证明：视图共享同一份底层数据

### 写（20分钟）

- [ ] 用自己的话写下"表"和"视图"的区别（一句话能说清吗？）
- [ ] 记录你觉得看板视图适合什么场景
- [ ] 明天的目标：画册视图 + 筛选初体验

## Day 6（周六）—— 画册视图 + 筛选初体验

### 拆（20分钟）

- [ ] 在模板中心找一个带"画册视图"的模板（比如产品目录、内容库）
- [ ] 观察：
  - 画册视图用什么字段作为"封面"？
  - 卡片上显示了哪些字段？为什么只显示这几个？
  - 这种视图适合什么场景？（浏览发现 vs 精确查找？）

### 建（60分钟）

- [ ] 创建 **视图3：画册视图**
  - 选择显示字段：资源标题、资源类型、主题标签、质量评分
  - 如果有资源是有封面图的（比如视频缩略图），可以加一个"封面"附件字段
  - 体会：同样的数据，画册视图的"浏览感"完全不同

- [ ] 学习**筛选**的基本操作：
  - 在表格视图中，添加筛选条件："状态 = 未看"
  - 观察：只剩下未看的记录了
  - 再加一个条件："资源类型 = 视频"
  - 观察：AND条件 → 范围更小了
  - 试试把AND改成OR → 结果有什么不同？

- [ ] 创建一个**带筛选的保存视图**：
  - "待学习视频"视图 = 状态为"未看" + 类型为"视频"
  - 保存后，这个视图随时打开就是这个结果

### 写（20分钟）

- [ ] 三种视图（表格/看板/画册）各适合什么场景？列一个对比
- [ ] 筛选的本质是什么？（用自己的话）
- [ ] 明天的目标：做一次小回顾 + 周总结

## Day 7（周日）—— 本周回顾 + 查漏补缺

### 拆（20分钟）

- [ ] 最后再打开一个你没看过的模板（任意类型）
- [ ] 计时！10分钟内尝试回答：
  - 有几个字段？各是什么类型？
  - 主键是什么？
  - 有几种视图？每种视图的用途是什么？
  - 你觉得这个模板设计得好不好？为什么？
- [ ] 这就是你第1个"能力信号"的练习：看懂任何一个表

### 建（60分钟）

- [ ] 检查你的"学习资源库"：

  - 单选/多选的选项是否统一？（有没有"Python"和"python"并存的情况？）
- [ ] 做一次"数据清洗"：
  - 统一选项命名
  - 填充明显的空值
  - 删除无效/重复记录
- [ ] 最终确认3个视图都能正常工作：
  - 表格视图（按时间降序）
  - 看板视图（按状态分组）
  - 画册视图（浏览发现用）

  - 本周学会了哪些概念？（用自己的话各说一句）
  - 操作上还有哪里不熟？（标记出来，下周顺手练）
  - 最大的认知变化是什么？
  - 给自己打个分（1-10）：对"字段类型"的理解
  - 给自己打个分（1-10）：对"视图"的理解
  - 给自己打个分（1-10）：对"主键"的理解

- [ ] 下周预告：第2周开始学"关联"——你会建第二张表，让两张表对话

## 常见卡点 & 解法

| 卡点 | 解法 |
|------|------|
| 不知道书签怎么导出 | Chrome: 设置 → 书签和列表 → 书签管理器 → 右上角⋮ → 导出书签 |
| 不确定该用什么字段类型 | 问自己：这个信息是"从几个选项中选一个"还是"随便填"？前者=单选，后者=文本 |
| 纠结字段设计"完不完美" | 不用完美，第4周就是来推翻重建的。先有，再好 |
| 数据量凑不到50条 | 把你微信收藏、B站收藏、知乎收藏都翻一遍，肯定够 |
| 看板视图不知道拿什么分组 | 用"状态"字段（未看/在看/已看）是最经典的看板分组 |
